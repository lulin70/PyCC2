## 1. ゲーム概要

### 🎯 これはどんなゲーム？

PyCC2は**リアルタイムタクティクス（RTT）**ジャンルのゲームです。アクションゲームのような素早い反射神経よりも、**戦術的な判断力**が勝利の鍵を握ります。

### ⚔️ 勝利条件

| 条件 | 説明 |
|------|------|
| **殲滅勝利** | 敵軍ユニットを全て撃破する |
| **目標占領** | マップ上のObjective（目標地点）を一定時間占拠する |
| **敗北条件** | 自軍が全滅する、または制限時間内に目標達成に失敗する |

### 🎮 ゲームモード

- **チュートリアル**: 基本操作を学ぶ10×10マップ
- **スキルミッション**: 単一マップでの戦闘
- **キャンペーン**: マーケット・ガーデン作戦全3ミッション（経験値・損害を引き継ぎ）

### 🗺️ 地形活用の極意

#### 攻撃側の地形戦術

```
👉 【開地からの突撃】
   道路上を移動して迅速に接近 → 草地で散開 → 敵の側面から攻撃

👉 【森林の活用】
   森林 = 視線遮断 + 掩体 + 隠蔽 の三重メリット！
   接近してから不意打ちを仕掛けるのに最適

👉 【建物制圧】
   進入可能な建物は最高の防御拠点（掩体50%）
   ただし、手榴弾や迫撃炮には注意！
```

#### 防御側の地形戦術

```
🛡️ 【高地・建物の確保】
   固体建物や壁はLOSを完全に遮断 → 敵の射線を封じられる

🛡️ 【弾坑の利用】
   爆撃跡は意外なほど良い掩体（25%）
   迫撃炮着弾点が自然と「塹壕」になる

🛡️ 【湿地・浅瀬】
   移動コストが非常に高い（湿地4.0x）
   敵の進行速度を大幅に鈍らせる天然の障害物
```

## 4. 戦闘メカニクス

### 🔫 弾道計算エンジン (BallisticEngine)

PyCC2のコアとなる戦闘計算システムです。すべての射撃は以下のステップで処理されます：

```
① 射程チェック → ② 視線(LOS)判定 → ③ 命中率計算 → ④ ダイスロール → ⑤ ダメージ計算 → ⑥ 抑制付与
```

#### 命中率の決定要因

| 要因 | 効果 | 説明 |
|------|------|------|
| **基礎命中率** | 武器依存 | ラフル:85% / SMG:70% / スナイパー:95% / MG:75% |
| **距離減衰** | 距離÷有効射程 × 0.6 | 遠くなるほど命中率が低下 |
| **掩体修正** | 1 - 掩体ボーナス | 森林で-20%、建物で-50% |
| **士気修正** | 0.3 ~ 1.0 | 低士気の敵は当たりやすくなる |
| **最終範囲** | 2% ~ 98% | 最低・最高命中率のキャップ |

#### ダメージ計算式

```
ダメージ = 基礎ダメージ × 貫通力 × 距離係数 × 乱数(0.8~1.2)
```

- **乱数幅 ±20%**: 同じ条件でもダメージにばらつきがあります
- **キリングブロウ確率 15%**: HPが0以下になった際、即死判定が発生
- **被弾角度**: 側面・背面からの攻撃は追加ダメージ

#### 武器性能比較

| 武器 | 基礎命中 | 有効射程 | 基礎ダメージ | 貫通力 | 散布角 |
|------|---------|---------|------------|--------|--------|
| M1ガーランド（ラフル） | 85% | 12 | 25 | 1.0 | 2.0° |
| トンプソン（SMG） | 70% | 6 | 15 | 0.8 | 4.0° |
| M1903（スナイパー） | 95% | 20 | 50 | 1.5 | 0.5° |
| MG42（機関銃） | 75% | 15 | 30 | 1.2 | 3.0° |
| M1911（ピストル） | 65% | 4 | 12 | 0.6 | 3.5° |

### 💥 抑制システム (Suppression)

外れた弾も無意味ではありません！**抑制値**が蓄積することで、敵を機能不全に陥れることができます：

| 抑制レベル | 蓄积量 | 効果 |
|-----------|--------|------|
| **通常** | 0〜49 | 通常状態 |
| **抑制状態 (SUPPRESSED)** | 50+ | 命中率-30%、移動速度低下 |
| **釘付け (PINNED)** | 80+ | 射撃・移動ほぼ不可能 |
| **恐慌 (PANIC)** | 100+ | 制御不能、潰走開始 |

> 💡 **戦術的応用**: 機関銃分隊で敵を「釘付け」にし、その隙に歩兵分隊で側面から襲撃する — これこそが現代歩兵戦術の基本です！

### 📐 視線 (Line of Sight)

**Bresenhamアルゴリズム**による正確な視線計算：

- **森林・固体建物・水域・壁** → 視線を**完全に遮断**
- **草地・荒地・生垣** → 視線は通るが、**隠蔽補正**が適用
- **高さ差** → 将来的に実装予定（現在は平面）

> 🎯 **スナイパーの極意**: 視線が通るギリギリの位置から射撃すると、**隠蔽補正**を受けずに一方的に攻撃できます！

## 5. 士気システム

### ❤️ 士気とは？

士気はユニットの**心理状態**を数値化したものです（0〜100）。士気が低いと命中率が下がり、最悪の場合は**恐慌状態**になって制御不能になります。

### 士気レベルと効果

| レベル | 値域 | 状態名 | 効果 |
|--------|------|--------|------|
| 🟢 **高昂** | 80-100 | High | 命中率+10%、移動+5% |
| 🟡 **平常** | 50-79 | Normal | 補正なし |
| 🟠 **動揺** | 30-49 | Shaken | 命中率-10% |
| 🔴 **恐慌** | 10-29 | Panic | **制御不能**、ランダム移動 |
| ⚫ **潰逃** | 0-9 | Rout | **指揮系統から離脱**、マップ外へ逃走 |

### 士気に影響するイベント

| イベント | 士気変化 | 説明 |
|---------|---------|------|
| 🫂 友軍指揮官が近くにいる | +10/tick | 指揮官のプレゼンス効果 |
| 💀 分隊長（リーダー）死亡 | -30 | 編成崩壊の衝撃 |
| 💀 友軍ユニット死亡 | -15 | 同僚を失う悲しみ |
| 😵 被弾（ダメージ受ける） | -5〜-20 | ダメージ量に比例 |
| 😰 抑制を受ける | -5/回 | 心理的圧力 |
| 🏳️ 指揮官死亡 | -25（全編成） | 全体的な士気喪失 |
| 🧘 戦闘から離れる | +2/tick | 自然回復 |
| 🎯 勝利的な交戦 | +5/kill | 戦果による高揚 |

> 💡 **重要**: 指揮官ユニットは「士気アンカー」です。最前線に配置して友軍の士気を維持しましょう！

### 恐慌の伝染

⚠️ **注意**: 恐慌状態のユニットは**近くの味方にも悪影響**を及ぼします。一人の恐慌が連鎖反応を起こし、全線崩壊につながることがあります！

**対策**:
- 恐慌ユニットを**早期に後退**させる
- 指揮官を**近づけて**士気回復を促す
- 衛生兵で**HPを回復**させて安心感を与える

## 6. 視界とフォッグ・オブ・ウォー

### 🌫️ FoW（战争迷雾）とは？

FoWは、**自軍ユニットが実際に「見えている」範囲のみ**を表示するシステムです。これにより、偵察の重要性とサプライズ要素が生まれます。

### 3種類の視界状態

| 状態 | 表示 | 説明 |
|------|------|------|
| 👁️ **視界内** | 明るく表示 | 現在見えている範囲。敵ユニットも確認可能 |
| 🔍 **既探索** | 半透明（暗い） | かつて見た場所。地形は見えるが、敵の動きはわからない |
| 🌑 **未探索** | 完全に黒 | 一度も見ていない領域。何があるか不明 |

### FoWの更新頻度

- **更新間隔**: 3〜5フレームごと（パフォーマンス最適化）
- **視界形状**: 円形（各ユニットの視界範囲による）
- **相互性**: **敵軍も同じFoWルール**に従います

### 🕵️ 偵察戦術

```
【基本原則】
1. スナイパーチームを先頭に配置 → 広視界で索敵
2. 歩兵分隊を左右に展開 → 視界カバー範囲を最大化
3. 森林の縁沿いに進む → 自身は隠れながら敵を発見

【高度テクニック】
・「覗き撃ち」: 森林の端から一時的に出て敵を確認し、すぐ引っ込む
・「音響偵察」: 敵の射撃音で位置を特定（将来的実装）
・「誘饵」: わざと一部隊を見せて、敵の配置を探る
```

## 7. AI敵対システム

### 🤖 AIの意思決定：ビヘイビアツリー (BT)

敵AIは**ビヘイビアツリー**という階層的決定構造を使用しています。AIは以下のように思考します：

```
🧠 AI思考プロセス
├── 敵を発見しているか？
│   ├── YES → 脅威度評価
│   │   ├── 高脅威 → 優先攻撃目标選択
│   │   └── 低脅威 → 無視 or 監視継続
│   └── NO → パトロール / 待機
├── HPは十分か？
│   ├── YES → 継続交戦
│   └── NO → 掩体へ退避 or 撤退
└── 士気は十分か？
    ├── YES → 通常行動
    └── NO → 恐慌 / 潰逃（プレイヤーと同じルール！）
```

### AIの特徴と対策

| AI挙動 | 説明 | 対策 |
|--------|------|------|
| **優先目标選択** | 視界内で最も脅威度の高い目标を攻撃 | 指揮官を隠す、囮を使う |
| **A*経路探索** | 最短経路で移動、壁を通り抜けない | 経路予測して待ち伏せ |
| **意思決定クールダウン** | 500msごとに再評価（毎フレームではない） | 行動の急変は起きにくい |
| **協調行動** | 複数ユニットが重ならないように配置 | 包囲されやすい、分散を心掛ける |
| **FoW制限** | 未発見のプレイヤーは攻撃できない | FoWを味方につける |
| **士気影響** | AIも恐慌・潰逃する | 指揮官を狙って士気を崩す |
| **迫撃炮要請** | 間接火力で遮蔽位置を攻撃 | 位置を頻繁に変える |

### 🎯 AI難易度システム

| 難易度 | AI反応速度 | 命中補正 | 備考 |
|--------|-----------|---------|------|
| **Easy** | 遅い (1000ms) | -20% | 初心者向け |
| **Normal** | 標準 (500ms) | ±0% | 推奨設定 |
| **Hard** | 早い (250ms) | +15% | ベテラン向け |
| **Veteran** | 即座 (100ms) | +30% | 達人向け |

## 9. 戦術指南

### 🏆 勝利への黄金則

#### ❶ 「森林こそ最大の味方」

森林は**三重のメリット**を提供します：
- 視線を遮断（敵に見られない）
- 20%の掩体ボーナス（被弾率低下）
- 50%の隠蔽補正（発見されにくい）

> **鉄則**: 開地での戦闘は最後の手段。常に森林や建物を利用した戦いを目指しましょう。

#### ❷ 「数の法則」を味方につける

1vs1の正面衝突は避け、**局地的な数的優位**を作り出します：

```
悪い例:
  [敵MG] ←→ [我方歩兵]   ← 1vs1で正面から挑む

良い例:
       [我方歩兵A]
             ↘
  [敵MG] ←←← [我方歩兵B]   ← 2方向から包囲攻撃
             ↗
       [我方スナイパー]
```

#### ❸ 「指揮官を守れ」

指揮官の死亡 = **全編成に-25の士気ペナルティ**。これは致命傷になり得ます。

- 指揮官を**最前列に出さない**
- 歩兵分隊を**盾**として配置
- 指揮官の**士気回復範囲（半径3タイル）**内に友軍を配置

#### ❹ 「抑制から殲滅へ」

直接ダメージを与える前に、まず**抑制**で敵を無力化します：

```
フェーズ1: MG分隊で敵をSUPPRESSEDに → 命中率-30%
フェーズ2: スナイパーで指揮官を排除 → 士気-25
フェーズ3: 歩兵分隊で側面から殲滤 → 被弾率最小化
```

#### ❺ 「スナイパーの隠密工作」

スナイパーチームの真の力は**一方的な攻撃**にあります：

1. 森林の**端**に潜伏（視線は通るが隠蔽補正あり）
2. 敵の**視界外**から狙撃（FoWを味方につける）
3. **指揮官**や**機関銃手**など高価値目標を優先
4. 位置が露見したら即座に移動

#### ❻ 「時間は味方ではない」

PyCC2は**リアルタイム**です。考え込んでいる隙に前線が崩壊することがあります：

- **ポーズ（Space）**を恐れず使う
- 重要な命令は**事前に計画**
- 多同時操作が必要なら**グループ化**（Squad）を活用

### 🎓 上級テクニック

#### A*経路読み

AIはA*アルゴリズムで最短経路を探索します。つまり：

- **橋**や**道路**はAIも優先的に使う → **待ち伏せポイント**に最適
- **湿地**や**荒地**はAIも避ける → **迂回路**を封じると強制ルートを作れる
- **森林**はAIの視界も遮る → **森の出口**で迎撃

#### 士気コンボ

```
1. 指揮官を前線に → 友軍士気+10/tick（維持）
2. 敵指揮官をスナイプ → 敵全体-25（崩壊起点）
3. MGで抑制 → 恐慌状態へ追い込み
4. 潰逃開始 → 連鎖的崩壊 → 勝利 ✓
```

#### 弾道計算を理解して使いこなす

- **距離×0.6÷有効射程** = 減衰率 → 有効射程ギリギリは命中率約40%
- **散布角×√距離** = 弾のばらつき → 遠距離ほど外れる可能性大
- **貫通力×距離係数** = 実ダメージ → 近距離の方がダメージ大

## 10. キャンペーンモード

### 🎖️ マーケット・ガーデン作戦

1944年9月17日〜25日、連合軍はオランダ深くへ空挺部隊を降下させ、ドイツ軍の防衛線を突破しようとしました。PyCC2のキャンペーンはこの歴史的作戦を3つのミッションで再現します。

### ミッション一覧

| ミッション | 名称 | 地図 | 目標 | 難易度 |
|-----------|------|------|------|--------|
| **Mission 1** | Eindhoven Bridge | ユインホーフェン橋 | 橋頭堡の確保 | ★★☆☆☆ |
| **Mission 2** | Nijmegen Crossing | ナイメーヘン渡河 | ワール川渡河点の制圧 | ★★★☆☆ |
| **Mission 3** | Arnhem Last Stand | アーネヘンの最後の一戦 | 橋梁防衛72時間 | ★★★★★ |

### 引継ぎシステム

ミッション間で以下のデータが引き継がれます：

- ✅ **生存ユニット** — 次のミッションでも使用可能
- ✅ **累積経験値** — ユニットの熟練度が向上
- ✅ **損害記録」— 重傷ユニットはHP減少した状態で参加
- ❌ **死亡ユニット** — 復活しません（永続的な損失）
- ❌ **弾薬消費」— 補給可能なミッションを除き引き継ぐ

### セーブシステム

- **8つのセーブスロット** — 好きなタイミングで保存可能
- **HMAC-SHA256署名** — セーブデータの改ざん検出
- **クイックセーブ/ロード** — F5/F9で即時保存・復帰
- **セーブメタ情報** — 時刻 / ミッション / 進捗概要を表示
