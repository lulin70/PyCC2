## クイックスタート

### 前提条件

- **Python 3.11+**（3.12+推奨）
- **Pygame 2.2+**
- **macOS / Linux / Windows**

### インストール

```bash
# リポジトリをクローン
git clone https://github.com/lulin70/PyCC2.git
cd PyCC2

# 仮想環境を作成
python -m venv .venv
source .venv/bin/activate   # macOS/Linux
# .venv\Scripts\activate    # Windows

# パッケージをインストール（開発用エディタブルモード）
pip install -e .

# または開発依存関係付きでインストール
pip install -e ".[dev]"
```

### ゲームの起動

```bash
# ゲームを起動
pycc2

# またはPythonモジュールとして起動
python -m pycc2.main
```

### 操作方法

| アクション | 入力 |
|-----------|------|
| ユニット選択 | 左クリック |
| コマンド発行 | 右クリックドラッグ（ラジアルメニュー）またはホットキー（Z/X/S/C/V/D/H） |
| 複数選択 | Shift + 左クリック |
| コマンドキュー | Shift + 右クリック |
| カメラ移動 | 矢印キー / WASD / エッジスクロール |
| ズーム | マウスホイール |
| 一時停止 | ESC（メニュー）/ Space（時間制御） |
| LOS確認 | Ctrl長押し |

## 技術アーキテクチャ

```
PyCC2/
├── src/pycc2/
│   ├── domain/              # コアゲームロジック（純粋Python、高テスト性）
│   │   ├── ai/             # ビヘイビアツリーAI、戦術決定システム
│   │   ├── components/     # ECS：Health、Morale、Weapon、Position、Vision
│   │   ├── entities/       # Squad、Unit、GameMap、Projectile
│   │   ├── systems/        # Campaign、Combat、Ballistics、Pathfinding
│   │   └── value_objects/  # Damage、Direction、TerrainType、Vec2
│   ├── services/           # ゲームループ、AIサービス、Event bus、Combat director
│   ├── presentation/       # レンダリング、入力処理、UI、オーディオ
│   │   ├── rendering/      # Camera、HUD、Minimap、Sprites、Isometricエンジン
│   │   │   ├── surface_pool.py          # 統合Surface LRUプール
│   │   │   ├── shell_casing_system.py   # 薬莢排出物理システム（125行）
│   │   │   ├── flash_effect_system.py   # 画面フラッシュエフェクトシステム（101行）
│   │   │   ├── weather_system.py        # 天候オーバーレイレンダリングシステム（160行）
│   │   │   └── fade_transition.py       # UIアルファフェードトランジション
│   │   ├── input/          # コマンドシステム、インタラクションコントローラ
│   │   ├── ui/             # メニュー、パネル、ツールチップ、展開UI
│   │   └── audio/          # サウンドシステム、音声コマンド
│   └── infrastructure/     # セーブシステム、設定、パーサー
├── data/
│   ├── maps/               # 63の史実マップJSONファイル
│   ├── scenarios/          # 11のシナリオ設定
│   └── units/              # ユニットテンプレート定義
├── tests/                  # 6536テスト（ユニット + 統合 + E2E + スモーク）
├── assets/                 # スプライト、サウンド、CC2リファレンススクリーンショット
└── docs/                   # 設計文書、PRD、ギャップ分析
```

**設計原則**：
- **ドメイン駆動設計**：ビジネスロジックとインフラストラクチャの明確な分離
- **イベント駆動アーキテクチャ**：システム間の疎結合のためのEventBus
- **固定タイムステップ**：ロジック@30 UPS、レンダリング@60 FPS
- **コンポーネントベースエンティティ**：柔軟なユニット構成のためのECSパターン
- **ビヘイビアツリーAI**：モジュラーで拡張可能なAI決定フレームワーク

コントリビュートを歓迎します！まだ多くの参加機会があるアーリーステージプロジェクトです：

1. **バグレポート** — ログ、スクリーンショット、再現手順を含めてください
2. **コード** — 既存のパターンに従い、テストを追加し、スタイルガイドを維持してください
3. **アセット** — スプライト、サウンド、マップは常に必要です（assets/README.mdを参照）
4. **ドキュメント** — ドキュメント、ユーザーガイド、チュートリアルの改善
5. **プレイテスト** — ゲームプレイバランスとCC2本格性に関するフィードバック

### 開発環境セットアップ

```bash
# 開発依存関係をインストール
pip install -e ".[dev]"

# コード品質チェック
ruff check src/ tests/          # リント
ruff format src/ tests/         # フォーマット
mypy src/pycc2/domain/          # 型チェック（ドメインレイヤー）

# プレコミットフックを実行
pre-commit run --all-files
```

完全なセットアップ手順は[INSTALL.md](INSTALL.md)を参照してください。
