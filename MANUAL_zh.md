## 1. 前言：欢迎来到战场

PyCC2 不是一款「点击即赢」的快餐游戏。它是一款 **实时战术模拟器（Real-Time Tactical Simulator）**，要求你像真正的连排级指挥官一样思考。

在这里，你不会看到血条从 100 滴到 0——你看到的是士兵们在弹雨中匍匐前进，是班长阵亡后全班的恐慌蔓延，是一发炮弹落在掩体外三米处引发的连锁崩溃。

**本手册将帮助你理解每一项机制，让你从一名菜鸟成长为能驾驭战场的合格指挥官。**

### 核心设计理念

| 理念 | 说明 |
|------|------|
| **真实性优先** | 弹道计算基于物理模型，而非简单的随机数 |
| **信息不对称** | 战争迷雾意味着你永远无法掌握全部情报 |
| **人性因素** | 士兵不是机器人——他们会恐惧、会犹豫、会英勇 |
| **战术深度** | 地形、队形、火力协同，每一个选择都有后果 |

## 2. 游戏背景：市场花园行动

**时间**：1944 年 9 月 17 日 — 9 月 25 日

**地点**：荷兰，从埃因霍温到阿纳姆的狭长走廊

**历史背景**：

1944 年夏末，盟军在欧洲西线的推进势头正劲。蒙哥马利元帅提出了一个大胆的计划——**市场花园行动（Operation Market Garden）**：动用空降部队夺取荷兰境内一系列关键桥梁，为地面装甲部队开辟一条直插德国鲁尔工业区的「高速公路」。

计划分为两个部分：
- **「市场」（Market）**：空降作战——美英波三国空降师分别夺桥
- **「花园」（Garden）**：地面推进——英国 XXX 军团的装甲部队沿公路北上驰援

然而，德军在阿纳姆附近意外集结了武装党卫军第 II 装甲军的精锐残部……历史告诉我们，这场行动最终以「一座桥之遥」的惨烈失败告终。

**你在 PyCC2 中扮演的角色**：盟军某连/排级指挥官，率领你的小队在荷兰的城镇、田野、森林和桥梁上执行战术任务。每一个决策都可能改变你手下士兵的命运。

## 4. 兵种介绍

PyCC2 中的每类单位都有其独特的定位和战术价值。没有「最强兵种」，只有「最合适的兵种」。

### 步兵班（Infantry Squad）

> **「步兵是战场的女王。」** — 传统军事格言

| 属性 | 数值 |
|------|------|
| 人数 | 10–12 人 |
| 主要武器 | M1 加兰德半自动步枪 |
| 副武器 | M1A1 汤姆逊冲锋枪（班长/副班长） |
| 反装甲能力 | ★☆☆☆☆（手榴弹仅对轻型载具有效） |
| 最佳射程 | 中距离（100–250 米） |
| 移动速度 | 中等 |

**战术定位**：多用途主力部队。适合占领建筑、控制路口、提供持续火力压制。是你在战场上最常依赖的单位。

**使用技巧**：
- 利用地形掩护接近目标，避免在开阔地长时间暴露
- 保持班组完整——伤亡过半后战斗力急剧下降
- 班长阵亡会导致士气大幅下跌，务必重点保护

---

### 机枪班（Machine Gun Squad）

> **「一挺机枪顶得上十支步枪——前提是你找得到好的射击位置。」**

| 属性 | 数值 |
|------|------|
| 人数 | 4–6 人 |
| 主要武器 | M1919A4 轻机枪 / M2HB 重机枪 |
| 射速 | 高（450–600 发/分钟） |
| 有效射程 | 远距离（300–800 米） |
| 移动速度 | 慢（需携带三脚架/弹药箱） |
| 弹药消耗 | 极高 |

**战术定位**：火力支援核心。一旦架设完毕，能在极短时间内倾泻大量弹药，形成有效的压制火力带。

**使用技巧**：
- **必须先架设再开火**：移动中的机枪精度极低且容易暴露
- 选择**交叉火力阵地**：两挺机枪互成角度布置，可形成致命杀伤区
- 注意**弹药管理**：机枪班的弹药消耗速度是普通步兵的 5–8 倍
- **高价值目标**：敌方 AI 会优先尝试消灭你的机枪班

### 狙击小组（Sniper Team）

> **「一颗子弹，一个目标。安静、精准、致命。」**

| 属性 | 数值 |
|------|------|
| 人数 | 2 人（狙击手 + 观察员） |
| 主要武器 | M1903 春田狙击步枪（配 M73 瞄准镜） |
| 射程 | 超远距离（500–1000 米） |
| 射速 | 极低（约 15–20 发/分钟） |
| 隐蔽性 | 极高（可进入潜行状态） |
| 发现概率 | 低（静止时几乎不可见） |

**战术定位**：精确打击与侦察先锋。用于清除敌方关键目标（机枪手、指挥官、反坦克手）或在前沿进行静默侦察。

**使用技巧**：
- **耐心是狙击手的最大武器**：不要急于开火，等待最佳射击时机
- 观察员的**标记能力**可以大幅提高命中率和发现隐藏敌人
- 狙击小组**极度脆弱**——一旦被近身几乎没有还手之力
- 配合其他单位使用：狙击手点掉机枪手后，步兵班再发起冲锋

### 医疗组（Medic Team）

> **「我们救不了所有人，但能救一个是一个。」**

| 属性 | 数值 |
|------|------|
| 人数 | 2–3 人（军医 + 卫生员） |
| 主要装备 | 医疗包、担架、急救用品 |
| 作战能力 | 极低（仅配备自卫用手枪） |
| 治疗范围 | 相邻格子或同一区域内 |
| 治疗效果 | 恢复轻伤员 HP，防止重伤员死亡 |

**战术定位**：战场上的生命线。虽然不具备正面作战能力，但一个及时到位的医疗组可以挽救整场战斗的走向。

**使用技巧**：
- **跟随主攻部队后方**，保持在安全距离内随时待命
- **优先救治班长和高价值单位**——他们的存活影响整个班组的效能
- 医疗组本身**非常脆弱**，需要其他单位的保护
- 在安全区域设立**临时救护站**可以让多个单位轮流接受治疗

### 坦克（Tank）

> **「当坦克的履带声响起时，步兵就知道——要么朋友来了，要么该跑了。」**

| 属性 | 数值 |
|------|------|
| 乘员 | 4–5 人（车长、炮手、装填手、驾驶员、副驾驶/机电员） |
| 主武器 | M4 75mm 坦克炮（谢尔曼） |
| 副武器 | M2HB 同轴机枪 + 车体机枪 |
| 装甲 | 正面厚 / 侧面中等 / 后方薄弱 |
| 对步兵威胁 | 极高（机枪扫射 + 炮击） |
| 易受威胁 | 反坦克武器、侧翼包抄、近距离炸弹 |

**战术定位**：机动火力堡垒。坦克的出现可以瞬间改变战场力量对比，但也可能成为敌方反坦克火力的集火目标。

**使用技巧**：
- **绝不要单独派遣坦克**——它需要步兵掩护侧翼和反坦克手
- 注意**装甲朝向**：始终让最厚的正面装甲朝向主要威胁方向
- 坦克的**观察能力受限**（观察孔视野狭窄），配合步兵侦察至关重要
- 在城镇战中要格外小心——**狭窄街道是反坦克伏击的绝佳场所**

## 5. 地形系统

地形是 PyCC2 中最重要的战术要素之一。**正确的地形利用可以将一支弱旅变成劲旅，而错误的地形选择会让优势兵力灰飞烟灭。**

### 地形类型一览

| 地形类型 | 图标 | 移动代价 | 掩体加成 | 视线遮挡 | 特殊效果 |
|----------|------|----------|----------|----------|----------|
| **开阔地**（Open Ground） | 🟩 | ×1.0 | 无 | 无 | 完全暴露，尽量避免 |
| **草地**（Grass） | 🌿 | ×1.0 | 轻微 (+10%) | 无 | 提供趴卧掩护 |
| **森林**（Woods） | 🌲 | ×1.5 | 强 (+40%) | 强 | 移动缓慢但隐蔽性极佳 |
| **城镇/建筑**（Urban/Building） | 🏠 | ×2.0 | 最强 (+60%) | 完全 | 室内战斗规则不同 |
| **弹坑**（Crater） | 💥 | ×1.2 | 中等 (+25%) | 轻微 | 炮击后的天然掩体 |
| **沼泽**（Swamp） | 🟫 | ×3.0 | 中等 (+20%) | 轻微 | 严重减速，车辆可能陷住 |
| **道路**（Road） | ══ | ×0.7 | 无 | 无 | 快速移动通道，但也是暴露通道 |
| **河流/桥梁**（River/Bridge） | 🌊 | — | 无 | — | 仅可通过桥梁渡河 |
| **高地/山丘**（Hill） | ⛰️ | ×1.3 | 中等 (+30%) | 提供 | 占据高处获得射程和视野优势 |

### 关键地形机制

#### 掩体加成（Cover Bonus）

当单位位于有掩体的地形时，其**被命中率**会按以下公式降低：

```
最终命中率 = 基础命中率 × (1 - 掩体加成百分比)
```

例如：基础命中率 70% 的射击，对位于森林中的目标实际命中率为：
`70% × (1 - 0.40) = 42%`

#### 视线遮挡（Line of Sight Blocking）

某些地形（如建筑、密集森林、高地）会**完全阻断视线**。这意味着：

- 你无法直接射击被遮挡的目标
- 敌人也同样看不到你
- **战争迷雾**会在这些区域产生「盲区」

#### 高地优势

占据高地的单位获得以下加成：

| 加成类型 | 效果 |
|----------|------|
| 射程延伸 | +20% 最大有效射程 |
| 视野扩展 | +30% 视野半径 |
| 命中率提升 | 对下方目标 +15% 命中率 |
| 反过来 | 攻击高地的敌人命中率 -10% |

## 6. 战斗机制

PyCC2 的战斗系统建立在**真实物理弹道模型**之上，而非简单的「掷骰子」判定。理解这些机制是在战场上取胜的关键。

### 6.1 弹道计算流程

每一次射击都会经历以下完整的计算管线：

```
[射手状态] → [弹道发射] → [飞行衰减] → [掩体判定] → [命中检定] → [伤害结算]
    ↓             ↓           ↓            ↓           ↓          ↓
 士气/疲劳      初速/散布   距离衰减     地形加成    概率命中    穿透/伤害
```

#### 各阶段详解

**① 射手状态评估**
- 士气过低 → 命中率惩罚（恐慌时命中率下降高达 50%）
- 疲劳累积 → 射速下降、散布增大
- 受伤状态 → 根据受伤部位施加不同惩罚

**② 弹道发射参数**
- 每种武器有其固有的**初速（Muzzle Velocity）** 和 **散布角（Spread Angle）**
- 距离越远，散布越大（呈二次函数增长）
- 移动中射击散布加倍

**③ 飞行衰减**
- 子弹能量随距离衰减（动能公式 `E = ½mv²`）
- 超过**有效射程**后，穿透力和伤害显著下降

**④ 掩体判定**
- 目标所在地形提供的**掩体加成**直接作用于最终命中率
- 处于「接敌姿态」的单位比「休息状态」的单位更难被命中

**⑤ 命中检定**
- 综合以上所有因子计算出**最终命中概率**
- 系统生成随机数进行判定
- **暴击机制**：有极小概率（~2%）造成双倍伤害

**⑥ 伤害结算**
- 命中后根据武器伤害值和目标护甲计算**实际伤害量**
- 不同身体部位的伤害倍率不同（头部 > 躯干 > 四肢）
- 致命伤害直接导致阵亡；非致命伤害导致**受伤状态**

### 6.2 伤害类型

| 类型 | 来源 | 效果 |
|------|------|------|
| **枪弹伤** | 步枪/机枪/冲锋枪 | 直接 HP 减少，可能致残 |
| **破片伤** | 手榴弹/迫击炮/炮弹 | 范围伤害，随距离衰减 |
| **挤压伤** | 坦克碾压/建筑物倒塌 | 高伤害，通常致命 |
| **震慑伤** | 近距离爆炸冲击波 | 短暂眩晕，士气骤降 |

### 6.3 压制效果（Suppression）

持续的火力压制会对目标单位产生以下效果：

| 压制等级 | 表现 |
|----------|------|
| **轻度压制** | 命中率 -15%，移动速度略降 |
| **中度压制** | 命中率 -35%，无法奔跑，只能爬行移动 |
| **重度压制** | 命中率 -60%，无法移动，只能就地还击 |
| **完全压制** | 无法还击，士兵蜷缩在地，士气快速下降 |

> **战术要点**：用机枪实施压制后，再派步兵从侧翼包抄——这是最基本的战术配合之一。

## 7. 士气与指挥

### 7.1 士气系统概览

PyCC2 采用**动态多因子士气模型**，每个单位拥有独立的士气值（0–100），并实时受到战场态势的影响。

```
士气值区间:
  90-100 : 🟢 昂扬 — 全属性小幅加成，士兵主动寻找战机
  70-89  : 🔵 正常 — 标准作战状态
  50-69  : 🟡 不稳 — 命中率开始下降，可能出现犹豫行为
  30-49  : 🠈 慌乱 — 显著减益，部分士兵可能拒绝前进
  0-29   : 🔴 恐慌 — 失去控制，士兵四散奔逃或投降
```

### 7.2 士气影响因素

| 因素 | 影响 | 权重 |
|------|------|------|
| **伤亡比例** | 每损失一人，士气下降 | ⭐⭐⭐⭐⭐ 最高 |
| **班长/指挥官阵亡** | 一次性大幅扣减（-25 ~ -40） | ⭐⭐⭐⭐⭐ 最高 |
| **侧翼暴露** | 被敌人从侧翼/后方攻击时持续扣减 | ⭐⭐⭐⭐ 高 |
| **火力压制** | 处于被压制状态时逐渐下降 | ⭐⭐⭐ 中 |
| **友邻单位支援** | 附近有友军单位时逐渐恢复 | ⭐⭐⭐ 中 |
| **成功击杀敌人** | 小幅士气回升 (+3 ~ +8) | ⭐⭐ 低 |
| **完成战术目标** | 显著士气回升 (+10 ~ +20) | ⭐⭐⭐ 中 |
| **地形安全感** | 位于良好掩体中时缓慢恢复 | ⭐ 低 |
| **连续作战疲劳** | 随时间推移逐渐下降 | ⭐⭐ 低 |

### 7.3 恐慌与溃退

当士气降至 **30 以下** 时，单位进入**恐慌状态**：

- 士兵不再响应你的命令
- 他们会自动向「他们认为安全的方向」逃离
- 逃离过程中极易被追击杀伤
- **唯一恢复方法**：撤离交火区域足够长时间后，由军官重新整编

> **指挥官的第一法则**：预防恐慌远比治疗恐慌容易。时刻关注士气条，在崩盘前撤出危险单位。

### 7.4 指挥官影响力

你的角色作为战场指挥官，可以通过以下方式影响下属士气：

| 行为 | 效果 |
|------|------|
| **靠近指挥** | 指挥官单位相邻时，周围单位士气恢复速度 +50% |
| **下达激励指令** | 消耗指挥点数，短时间内提升目标单位士气 |
| **合理撤退** | 及时下令撤退避免无谓伤亡，长期来看有助于维持士气 |
| **达成目标** | 每完成一个战术目标，全军士气小幅提升 |

## 8. 战争迷雾与视野

### 8.1 什么是战争迷雾？

战争迷雾（Fog of War）模拟了真实战场上**信息不完全**的本质。你只能看到你的单位**实际观察到**的区域，其余部分都被迷雾笼罩。

### 8.2 迷雾等级

| 等级 | 显示 | 说明 |
|------|------|------|
| **已探索** | 正常亮度 | 当前可见，信息实时更新 |
| **已探明** | 半透明灰色 | 曾经见过但当前不在视野内（可能有变化！） |
| **未探索** | 完全黑色 | 从未被任何己方单位观察过的区域 |

> **致命陷阱**：「已探明」区域的信息可能是过时的！敌人可能已经移动到你上次看到的位置之外。**永远不要假设你知道迷雾后面有什么。**

### 8.3 视野机制

每个单位都有一个**视野扇区**（Vision Cone），由以下参数决定：

| 参数 | 说明 | 典型值 |
|------|------|--------|
| **视野半径** | 能看清的最远距离 | 6–12 格（取决于兵种和姿势） |
| **视野角度** | 扇区的宽度 | 90°–180°（正面视野宽，背后窄） |
| **更新频率** | 视野刷新间隔 | 每游戏 tick 更新一次 |

### 8.4 Bresenham 视线算法

PyCC2 使用经典的 **Bresenham 直线算法** 来判断两点之间是否有视线遮挡：

```
  射手(A)                    目标(B)
    ●━━━━━━━━━━━━━━━━🌲━━━●
         ← 视线被森林阻挡 →
    
  结果: A 无法直接看到 B
```

**遮挡物包括**：建筑物、密林、高地（背坡）、大型载具

**非遮挡物包括**：灌木丛、矮墙、铁丝网、其他步兵单位

### 8.5 反侦察手段

| 手段 | 效果 |
|------|------|
| **静止不动** | 大幅降低被发现概率 |
| **利用阴影/暗处** | 夜间或室内环境下进一步降低 |
| **烟雾弹** | 制造临时视线阻断区域 |
| **保持低姿** | 趴卧时的轮廓明显小于站立 |
| **噪音控制** | 避免不必要的射击暴露位置 |

## 10. 战术指南

> **以下内容来自实战经验总结。掌握它们，你就超越了 80% 的玩家。**

### 10.1 黄金法则

#### 法则一：永远不要在开阔地停留

开阔地是步兵的坟墓。除非你有绝对的空中/装甲优势，否则穿越开阔地时应采用**分段跃进**战术——从一个掩体冲刺到下一个掩体，每次冲刺不超过 3–5 秒。

#### 法则二：火力优于机动

在 PyCC2 中，**先开火的人往往赢得战斗**。当你发现敌人时，第一反应应该是立即开火压制，而不是试图调整到一个「完美位置」。完美的位置不重要——活着的射手才重要。

#### 法则三：侧翼就是一切

正面强攻一个有准备的阵地是自杀行为。永远寻找敌人的**侧翼或后方**——哪怕绕行多花两分钟，也比正面硬拼少牺牲一半人。

#### 法则四：保护你的班长

班长阵亡 = 该单位战斗力永久性下降 30–50%。把班长放在队伍后方，让他专注于指挥而不是射击。

### 10.2 经典战术编组

#### 🔫 「锤砧」编组（Hammer and Anvil）

```
        [敌方阵地]
             ↑
     [机枪班] ← 砧（固定火力压制）
    /
[步兵班] ← 锤（侧翼包抄）
```

**执行步骤**：
1. 机枪班在敌方正面就位，开始持续压制射击
2. 趁敌人被压得抬不起头时，步兵班从侧翼迂回
3. 步兵班到达侧翼射击位置后，机枪班转移火力或继续压制
4. 步兵班从侧翼发起突击，敌人腹背受敌

**适用场景**：攻打敌方驻守的建筑、路口或防线

---

#### 🔭 「侦察-打击」编组（Recon-Strike）

```
[狙击小组] ← 前沿静默侦察
      ↓ 发现目标
[机枪班] ← 远程精确清除关键威胁
      ↓
[步兵班] ← 主力跟进清剿残余
```

**执行步骤**：
1. 狙击小组部署在最前沿的有利位置，开启静默侦察模式
2. 观察员标记敌方高价值目标（机枪手、反坦克手、指挥官）
3. 狙击手逐个清除已标记目标
4. 敌方火力减弱后，机枪班和步兵班推进清场

**适用场景**：进攻前期的情报获取与弱点打击

---

#### 🏥 「滚动治疗」编组（Rolling Aid）

```
[步兵班 A] ← 第一梯队进攻
    ↓ 完成交换/受损
[医疗组] ← 就位治疗
    ↓
[步兵班 B] ← 替补 A 队继续推进
    ↓
[步兵班 A] ← 恢复后归队
```

**适用场景**：持久战或多阶段进攻任务。通过轮换保证前线始终有满编状态的部队。

### 10.3 防守战术

#### 交叉火力（Crossfire）

设置两个或以上的火力点，使它们的**射击扇区相互重叠**。闯入重叠区域的敌人将同时面对多个方向的火力。

```
  [火力点 A] ╲
              ╳ ← 致命交叉区
  [火力点 B] ╱
```

####纵深防御（Defense in Depth）

不只设一道防线，而是设置**多道梯次防线**：

```
第1道防线: 前哨警戒（少量兵力 + 早期预警）
    ↓ 敌人突破
第2道防线: 主防御地带（主力 + 重武器）
    ↓ 敌人突破
第3道防线: 最后据点（预备队 + 反击力量）
```

即使敌人突破了第一道防线，后续每道防线都会继续消耗他们。

#### 机动防御（Mobile Defense）

故意在正面示弱，诱敌深入，然后用**预备队从侧翼反击**。高风险高回报——如果预判正确，可以对敌人造成毁灭性打击。

### 10.4 城镇战要点

城镇环境是最复杂也最危险的战场：

| 要点 | 说明 |
|------|------|
| **室内外差异** | 室内近战射程极短但命中率极高；室外则相反 |
| **窗户是双刃剑** | 可以向外射击，但也会暴露自己成为靶子 |
| **拐角危险** | 每个拐角都可能藏着敌人，使用「切拐角」技术（peek & shoot） |
| **楼层优势** | 二楼及以上高度对一楼目标有显著命中率加成 |
| **街道是死亡陷阱** | 绝不在街道中央停留，始终贴墙移动 |
| **破门而入** | 进屋前先用烟雾弹或手榴弹「打招呼」 |

### 10.5 反坦克战术

如果你没有专用的反坦克武器，以下方法可以对抗敌方坦克：

| 方法 | 效果 | 风险 |
|------|------|------|
| **侧翼包抄** | 攻击装甲薄弱的侧面/后方 | ⭐⭐⭐⭐ 高风险 |
| **近距离炸弹** | 多枚手榴弹集中投掷 | ⭐⭐⭐⭐⭐ 极高风险 |
| **引诱至伏击圈** | 将坦克引入预设的反坦克阵地 | ⭐⭐⭐ 中风险 |
| **攻击履带** | 使坦克失去机动能力 | ⭐⭐⭐ 中风险 |
| **呼叫支援** | 使用预设的空中/火炮支援 | ⭐ 低风险（如果有支援可用） |

## 11. 存档系统

### 11.1 存档机制概述

PyCC2 的存档系统采用**双重安全保障**：

1. **HMAC-SHA256 数字签名**：确保存档文件不被篡改
2. **Pydantic 数据验证**：确保存档数据结构完整有效

这意味著：
- ✅ 你的存档不会被意外损坏
- ✅ 任何人无法通过修改存档来作弊
- ✅ 即使文件被外部程序修改，加载时会检测到并拒绝

### 11.2 存档操作

| 操作 | 方式 | 说明 |
|------|------|------|
| **快速存档** | 按 `F5` | 覆盖最近的快速存档槽位 |
| **快速读档** | 按 `F9` | 读取最近一次快速存档 |
| **手动存档** | 菜单 → 保存游戏 | 可命名、可选槽位的完整存档 |
| **手动读档** | 菜单 → 加载游戏 | 从存档列表中选择读取 |
| **自动存档** | 系统自动触发 | 每完成一个关卡/关键节点时自动保存 |

### 11.3 存档文件位置

```
~/saves/
├── save_slot_0.json       # 快速存档（F5/F9）
├── save_slot_1.json
├── save_slot_2.json
├── ...
└── save_slot_7.json
```

### 11.4 存档完整性校验

每次加载存档时，系统会自动执行以下检查：

```
加载流程:
  1. 读取文件 → 2. HMAC 签名校验 → 3. Pydantic 数据解析 → 4. 游戏状态恢复
  
  如果第 2 步失败 → ❌ "存档文件已被篡改，拒绝加载"
  如果第 3 步失败 → ❌ "存档数据损坏，无法读取"
  全部通过          → ✅ "存档加载成功"
```
