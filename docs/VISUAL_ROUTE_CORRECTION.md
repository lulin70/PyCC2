# PyCC2 视觉路线修正: 从等距投影回归正交俯视

> **日期**: 2026-05-24
> **触发**: 用户提供4张CC2原版截图，发现之前对CC2投影方式的判断错误
> **核心结论**: CC2使用**正交俯视(Orthographic Top-Down)**，不是等距投影。之前的isometric系列属于过度设计。
> **原则**: 先达到95% CC2还原度，再考虑超越。

---

## 一、错误诊断

### 1.1 之前的错误假设

```
❌ 错误假设: CC2使用45°等距投影(Isometric)
   → 导致创建了 isometric_transform.py, isometric_tile_generator.py,
     isometric_renderer.py, isometric_building_renderer.py 等整套等距管线
   → 菱形瓦片(64×32)、坐标变换公式、深度排序等全部基于错误前提
```

### 1.2 CC2实际视觉特征（基于4张原版截图）

| 特征 | 实际值 | 证据 |
|------|--------|------|
| **投影方式** | 正交俯视 (Orthographic Top-Down) | 道路水平/垂直、河流垂直、建筑正方形 |
| **Tile形状** | 正方形 (非菱形) | 所有建筑屋顶都是正方形 |
| **坐标轴** | X水平向右，Y垂直向下 | 十字路口完美正交 |
| **建筑立体感** | 屋顶+侧面阴影条(伪3D) | 大建筑的侧面是单色暗带，非真3D渲染 |
| **单位视角** | 纯俯视 + 像素级阴影 | 步兵/坦克从正上方看 |

### 1.3 CC2的"立体感"来源（非等距）

CC2看起来有立体感，但来源是：

1. **建筑**: 正方形屋顶 + 底部/右侧1-2px深色阴影条 → 模拟厚度
2. **单位精灵**: 本身绘制时有轻微明暗面（头部亮、脚部暗）
3. **地形高度**: 高地用更亮的颜色，低地用更暗的颜色（颜色渐变，非几何）
4. **阴影**: 单位/树木有简单的右下方向椭圆阴影

**这些都是2D伪3D技巧，不需要坐标变换。**

## 二、CC2精确视觉参数（从截图提取）

### 2.1 地形参数

```
Tile尺寸: 约48×48像素 (屏幕上测量)
地形类型与配色:
  草地(Grass):    #4C7C23 主色, #5A8E2B 高光, #3A6418 暗部
                  - 有细微纹理变化(非纯色填充)
                  - 边缘有轻微不规则过渡到其他地形
  泥土(Dirt):     #8B6914 主色, #9E7A20 高光
                  - 用于弹坑边缘、裸露地面
  道路(Road):      #6B5A40 深灰棕 (碎石路)
                  - 宽度约1个tile
                  - 与草地有明显边界
  河流(Water):    #3060A0 蓝, alpha约180(半透明)
                  - 有波纹动画(亮白短线)
                  - 岸边有过渡带
  树篱(Hedgerow): #2A5420 深绿, 不规则块状
                  - 约4-6px宽的不规则线条
  石墙(Wall):     #707070 灰, 2-3px宽线条
  弹坑(Crater):    #5A4628 凹陷中心, #7A6840 边缘高光
                  - 椭圆形, 直径约20px
```

### 2.2 建筑参数

```
建筑尺寸:
  小屋(Small House):  1×1 tile (~48×48px)
  中型建筑(Medium):   2×2 tile (~96×96px)
  大型建筑(Large):    3×3 tile (~144×144px)

建筑样式:
  屋顶: 正方形, 颜色区分类型
    红顶房(Red Roof):   #B83020 (民用)
    灰顶房(Gray Roof):  #505860 (工业/军事)
    棕顶(Brown):        #8B6040 (谷仓)
  
  伪3D效果:
    - 屋顶下方有2-4px深的阴影条(模拟墙面)
    - 阴影色 = 屋顶色的暗化版本(亮度-30%)
    - 无真正的侧面多边形渲染
  
  建筑内部细节:
    - 工业建筑: 屋顶上有编号(黄色大数字 "2", "3", "5")
    - 教堂: 可能有十字架(顶部小标记)
    - 损坏状态: 屋顶变暗 + 裂纹线 + 部分变黑
```

### 2.3 单位参数

```
步兵(Infantry):
  尺寸: 约18×22像素 (画布24×24, 含透明边距)
  视角: 纯俯视 (能看到头顶盔、肩背、背部)
  朝向: 8方向 (N/NE/E/SE/S/SW/W/NW)
  配色:
    盟军(US/UK/PL): OD绿 #4B5320 制服, #505850 头盔
    轴心(DE):        Field灰 #555D50 制服, #525850 头盔
  细节:
    - 头盔有高光点(1-2px亮色)
    - 武器可见(步枪长约12px)
    - 身体略呈椭圆形(非矩形)
    - 有方向性姿态差异(相邻方向>30%像素不同)

载具(Vehicle):
  坦克(Tank):
    尺寸: 约36×36像素 (中型), 44×44(重型)
    视角: 俯视, 可见炮塔轮廓 + 履带矩形 + 车体
    细节:
      - 炮塔在车体中央偏上(圆形/方形)
      - 炮管从炮塔伸出(长8-12px, 2px宽)
      - 履带为两侧矩形(各4-6px宽)
      - 车体有装甲板接缝线(1px深色)
    配色:
      盟军: 橄榄绿 #4A5D23 + 泥土斑点
      轴心: 德国灰 #4C4C4C + 黄色条纹(可选)

半履带车(Halftrack):
  尺寸: 约32×40px
  特征: 后部履带(单侧), 前部车轮(2个圆), 开放式货箱

吉普车(Jeep):
  尺寸: 约24×18px
  特征: 小型轮廓, V形挡风玻璃线
```

### 2.4 UI参数

```
底部面板(CC2 Bottom Panel):
  高度: 约120px (屏幕底部)
  背景: 深橄榄绿 #3A4030
  分区:
    左侧: 兵力列表(单位名称 + 状态图标)
    中间: 命令按钮区(7个命令)
    右侧: 选中单位详情(名称/血量/士气/弹药)
  字体: 等宽字体, 白色文字, 12-14px
  按钮: 3D凸起效果(亮边+暗边)

小地图(Minimap):
  位置: 右下角面板内
  尺寸: 约120×120px
  显示: 地形缩略图 + 单位点 + 视野范围框

攻击线(Attack Line):
  颜色: 绿色实线(确认前) / 红色(确认后)
  线宽: 2px
  端点: 圆形标记(射手/目标位置)
```

### 2.5 特效参数

```
爆炸(Explosion):
  核心: 黄→白圆形扩散(10→25px, 3帧)
  烟云: 灰色不规则形(持续15-20帧, 渐隐)
  屏幕: 相机抖动(2-3px, 0.15s)

枪口焰(Muzzle Flash):
  形状: 黄→橙→白椭圆(6×4px)
  持续: 1-2帧(0.03-0.06s)

命中火花(Hit Spark):
  数量: 3-5个黄/白点
  运动: 放射状外扩(速度2-3px/frame)
  持续: 3-5帧

烟雾(Smoke Grenade):
  形状: 灰白色大圆形(半径3-5tile)
  边缘: 不规则锯齿
  内部: 半透明(alpha=150-200)
  持续: 约3回合(45s游戏时间)
```

## 三、修正后的技术路线

### 3.1 当前状态评估

| 模块 | 当前状态 | 与CC2差距 | 优先级 |
|------|---------|----------|--------|
| 投影方式 | 正交(默认)+等距(可选) | ✅ 正交正确 | — |
| 地形渲染 | 程序化生成, 32×32 tile | ⚠️ Tile偏小, 纹理不足 | P0 |
| 建筑渲染 | 程序化生成, 正方形 | ⚠️ 缺少伪3D阴影条 | P0 |
| 步兵精灵 | PixelArtist3D程序化 | ⚠️ 方向差异不够明显 | P0 |
| 载具精灵 | PixelArtist3D程序化 | ⚠️ 细节不足 | P1 |
| UI面板 | CC2BottomPanel | ✅ 接近CC2 | — |
| 攻击线 | AttackLineSystem | ✅ 接近CC2 | — |
| 小地图 | Minimap | ⚠️ 缺少地形细节 | P2 |
| 特效 | AnimationSystem | ⚠️ 需要调参匹配CC2 | P1 |

### 3.2 修正路线图 (达到95% CC2还原度)

#### Phase A: 地形和建筑修正 (P0, 最高优先)

**目标**: 地形和建筑从截图上看与CC2无法区分

1. **Tile尺寸调整**: 32×32 → 48×48 (匹配CC2截图实测)
2. **地形纹理增强**:
   - 草地: 添加细微颜色变化(±10%亮度随机噪声)
   - 道路: 添加碎石纹理(小灰点随机分布)
   - 河流: 添加波纹动画(亮白短线)
   - 弹坑: 椭圆形凹陷 + 边缘高光环
3. **建筑伪3D**:
   - 正方形屋顶(已有)
   - 新增: 底部2-4px深色阴影条(模拟墙面厚度)
   - 新增: 工业建筑屋顶编号(可选)
   - 新增: 损坏状态(裂纹线+局部变暗)

#### Phase B: 精灵质量提升 (P0)

**目标**: 步兵/载具精灵接近CC2风格

1. **步兵改进**:
   - 确保8方向有明显差异(>30%像素不同)
   - 添加头盔高光点
   - 武器清晰可见(12px长步枪)
   - 身体椭圆形(非矩形)
2. **载具改进**:
   - 坦克: 清晰炮塔+炮管+履带+装甲缝
   - 半履带: 后履带+前轮+货箱
   - 吉普: V形挡风玻璃

#### Phase C: 特效调参 (P1)

**目标**: 战斗特效匹配CC2节奏和外观**

1. 爆炸尺寸/持续时间匹配CC2
2. 枪口焰颜色/大小匹配
3. 烟雾扩散速率匹配
4. 屏幕抖动幅度匹配

#### Phase D: 外部资源集成 (P1)

**目标**: 用PixVoxel CC0资源替换程序化精灵**

1. 执行 `python scripts/download_pixvoxel_assets.py` (网络好时)
2. PixVoxelLoader加载正交版精灵(非等距版)
3. 调色板换色匹配CC2配色

## 四、Isometric模块处置方案

### 4.1 不删除的原因

1. 已投入大量工作(7个文件, 66个测试)
2. 作为"实验性功能"保留有价值
3. 未来如果要做"CC2 Mod"或"CC3风格"可以启用

### 4.2 处置方式

```
isometric_transform.py       → 保留, 标记 @experimental
isometric_tile_generator.py  → 保留, 标记 @experimental  
isometric_depth_sorter.py    → 保留, 标记 @experimental
isometric_renderer.py        → 保留, 标记 @experimental
isometric_building_renderer.py → 保留, 标记 @experimental
pixvoxel_loader.py           → 保留(通用加载器, 不限于等距)
camera.py ProjectionMode     → 保持, 默认ORTHOGRAPHIC
interaction_controller I键  → 保留(切换功能)
```

### 4.3 文档更新

- ISOMETRIC_ARCHITECTURE_PROPOSAL.md → 标记 "SUPERSEDED: CC2 uses Orthographic"
- TECH_DEBT.md TD-044 → 更新为 "等距模式作为实验性功能保留"
- VISUAL_SPEC.md → 重写为基于正交俯视的CC2还原规范
