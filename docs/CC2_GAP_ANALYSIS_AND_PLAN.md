# PyCC2 ↔ Close Combat 2: A Bridge Too Far 完整机制清单与差距分析

> **版本**: v3.0 | **日期**: 2026-05-26
> **分析方法**: CC2官方手册 + Grokipedia百科 + 代码审计 + 诚实评估
> **目标**: 从当前 ~45% 实际保真度提升至 **95%+ CC2 原版保真度**
> **原则**: 先接近CC2的水平，达到95%以上再考虑超越

---

## 一、诚实保真度重评估

### 1.1 评估方法说明

v1.0文档声称66%保真度，但存在以下问题：
- 将"有代码"等同于"已实现"，忽略了大量系统未集成到游戏循环
- 命令数量错误（7个含Sneak，CC2实际只有6个）
- 未评估各系统的实际运行效果

### 1.2 修正后保真度评估

| **交互方式错误** | 2 | 命令执行方式(无拖拽)、LOS检查(无Ctrl键) |

### 4.2 集成断裂清单

```
Unit实体缺少以下字段/引用:
  ├── fatigue: FatigueComponent     ← 存在但未挂载
  ├── veterancy: VeterancyComponent ← 存在但未挂载
  ├── crew: VehicleCrew             ← 存在但未挂载
  ├── squad: Squad                  ← 仅squad_id松耦合
  └── weather修饰未在tick中调用      ← 存在但未调用

游戏循环缺少以下调用:
  ├── FatigueComponent.accumulate() / recover()
  ├── VeterancyComponent accuracy/morale修正
  ├── WeatherEffects.apply_to_*()
  ├── 建筑室内模式自动切换逻辑
  └── Ctrl键LOS检查
```
