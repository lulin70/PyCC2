## 设计原则

### 核心原则: "像CC2，不像少儿游戏"

**绝对禁止**:
❌ 纯几何形状(圆/三角/方块)作为单位精灵
❌ 抽象符号表示战斗单位
❌ 卡通化/夸张比例
❌ 单色填充无细节
❌ 部件分解图（非游戏内使用的整合视图）

**必须做到**:
✅ 像素级人体/载具俯视图
✅ 可辨识的解剖结构(头/身/四肢/武器)
✅ 8个方向有明确的朝向差异
✅ 符合军事写实的色调和比例
✅ 32×32或48×48像素内的精细刻画
✅ 整体化的单帧精灵（非分解部件）

## 一、步兵单位精灵规格 (Infantry Unit Sprites)

### 1.1 基础人体模型 (Top-Down View)

```
像素布局示意 (32×32, 头部朝上/North):

     ↑ North (0°)
     ┌─────────────┐
     │    ╭───╮    │  ← 头部/头盔 (6×4px)
     │   │ ○ ○ │   │  ← 眼睛或头盔细节
     │    ╰───╯    │
     │   ╱││╲     │  ← 肩膀/背包轮廓
     │  ╱ ││ ╲    │
     │ │  ││  │   │  ← 躯干 (8×10px)
     │ │  ││  │   │
     │ ╰──┼──╯   │  ← 腰带/装备
     │  ╱    ╲    │
     │ ╱      ╲   │  ← 双腿 (各4px宽)
     │▓       ▓   │  ← 双脚/靴子
     │             │
     │     ││     │  ← 步枪 (2px宽)
     │     ││     │
     └─────────────┘

总高度: ~28px (头部到脚)
总宽度: ~14px (肩膀到肩膀)
颜色: 盟军=OD绿(#4B5320) / 轴心=Field灰(#555D50)
```

### 1.2 8方向姿态差异 (Critical!)

每个方向必须有**明显且可辨识**的差异:

#### North (N, 0°) - 面向屏幕上方
```
头部在顶部, 枪指向上方
可以看到: 头盔顶部、肩背、背部、腿后跟
```

#### Northeast (NE, 45°) - 面向右上
```
头部偏右上, 身体倾斜45°
枪指向右上方
可以看到: 右侧脸/头盔边缘、右肩、斜背
```

#### East (E, 90°) - 面向屏幕右方
```
头部在右侧, 身体水平
枪指向右方
可以看到: 右侧轮廓、持枪手臂
```

#### Southeast (SE, 135°) - 面向右下
```
头部偏右下, 身体倾斜135°
枪指向右下方
```

#### South (S, 180°) - 面向屏幕下方 (面向玩家)
```
头部在底部, 可以看到正面
枪指向下方(朝向玩家)
可以看到: 面部/头盔正面、胸部、双腿正面
⚠️ 这个方向应该最"对称"和"正面感"
```

#### Southwest (SW, 225°) - 面向左下
```
头部偏左下, 身体倾斜225°
枪指向左下方
```

#### West (W, 270°) - 面向屏幕左方
```
头部在左侧, 身体水平
枪指向左方
可以看到: 左侧轮廓
```

#### Northwest (NW, 315°) - 面向左上
```
头部偏左上, 身体倾斜315°
枪指向左上方
```

**关键要求**:
- 相邻方向差异必须肉眼可辨(>30%像素不同)
- N/S方向应该是近似镜像(但武器位置可能不同)
- E/W方向也应该是近似镜像

### 1.3 动画帧规范

#### 待机动画 (Idle, 2帧循环)
```
Frame 1: 标准站立姿势, 双腿微张
Frame 2: 重心轻微左移, 左腿稍曲 (模拟呼吸)
周期: 0.8秒 (慢节奏)
```

#### 行走动画 (Walk, 4帧循环)
```
Frame 1: 左脚前迈, 右臂前摆
Frame 2: 双脚并拢, 重心最高
Frame 3: 右脚前迈, 左臂前摆
Frame 4: 双脚并拢, 重心最高
周期: 0.4秒 (快节奏)
特点: 腿部有明显开合, 手臂协调摆动
```

#### 射击动画 (Shoot, 3帧一次性)
```
Frame 1: 举枪瞄准 (枪口前伸)
Frame 2: 枪口闪光+后坐力(身体微后仰)
Frame 3: 恢复待机
周期: 0.3秒
特效: Frame 2叠加黄色小椭圆(枪口焰, 4×3px)
```

#### 死亡动画 (Die, 4帧一次性)
```
Frame 1: 受击反应(身体歪斜, 闪红)
Frame 2: 倒地过程(身体倾斜45°)
Frame 3: 躺平地面(横向放置)
Frame 4: 尸体不动(变暗灰色)
周期: 0.6秒
最终: 保持Frame 4作为尸体标记
```

### 1.4 特殊兵种变异

#### 机枪手 (MG Team)
```
基础: 同步兵模型
变异:
  - 武器改为MG42/M1919机枪(更粗更长, 3×8px)
  - 可能增加弹药箱/三脚架(脚边小矩形)
  - 身体姿势: 半跪或卧姿(腿部弯曲不同)
颜色: 深绿色调 (+20%饱和度)
```

#### 反坦克兵 (AT Team)
```
基础: 同步兵模型
变异:
  - 武器: 巴祖卡火箭筒(粗短, 4×6px)
  - 背部: 大型背包(圆形凸起, 4×4px)
  - 姿势: 半蹲(重心更低)
颜色: 橙褐色调
```

#### 军官 (Officer)
```
基础: 同步兵模型
变异:
  - 头部: 不同头盔样式(贝雷帽/军官帽轮廓)
  - 肩部: 肩章标识(小横杠或星号, 2×2px)
  - 可能手持望远镜/手枪而非步枪
颜色: 金色/棕色点缀
```

#### 侦察兵 (Scout)
```
基础: 同步兵模型
变异:
  - 身体更瘦长(宽度-15%)
  - 可能带望远镜/双筒镜
  - 姿势更低(半蹲潜行态)
颜色: 浅绿/灰绿迷彩感
```

#### 工兵 (Engineer)
```
基础: 同步兵模型
变异:
  - 背部: 工具包/炸药包(方形凸起)
  - 腰部: 工具腰带(多点装饰)
  - 可能持铲子而非步枪
颜色: 蓝灰色调
```

#### 医疗兵 (Medic)
```
基础: 同步兵模型
变异:
  - 臂部: 红十字臂章(2×2px红色白底)
  - 背部: 医疗包(白色小包)
  - 武器: 卡宾枪/手枪(较小)
颜色: 同步兵+红色点缀
```

## 二、载具精灵规格 (Vehicle Sprites)

### 2.1 中型坦克 (Medium Tank, e.g., Sherman/Panzer IV)

```
尺寸: 48×48像素 (比步兵大50%)
视角: 纯正上方俯视

布局示意 (炮塔朝北):
        ┌─────────────────┐
        │    ╭─────╮      │  ← 炮塔 (16×14px)
        │    │  ○  │      │  ← 炮管口/舱盖
        │    │ ══╤ │      │  ← 炮管 (长度10px, 宽2px)
        │    ╰──┼─╯      │
        │   ╔══╧══╗       │  ← 车体上部 (28×20px)
        │   ║  ▓  ║       │  ← 舱盖/装甲板纹理
        │   ║  ▓  ║       │
        │   ╚══╤══╝       │
        │  ═══╧════       │  ← 履带 (两侧, 各6px宽)
        │ ░░░░░░░░░ │       │  ← 履带轮纹理
        │  ═══╧════       │
        └─────────────────┘

关键特征:
- 炮塔旋转独立于车体 (8方向 = 炮塔朝向)
- 炮管清晰可见 (长度>车身半径)
- 履带有轮子纹理 (3-4个圆形表示路轮)
- 车体有装甲板接缝线条 (深色1px线)
- 乘员舱盖可显示开启状态

颜色:
  盟军: 橄榄绿 (#4A5D23) + 泥土色斑点
  轴心: 德国灰 (#4C4C4C) + 黄色条纹 (可选)
```

### 2.2 载具8方向差异

与步兵类似, 但主要体现为**炮塔和车体朝向**:

| 方向 | 炮塔位置 | 炮管指向 | 车体细节 |
|------|---------|---------|---------|
| North | 顶部中央 | 向上 | 可见发动机舱盖 |
| East | 右侧 | 向右 | 可见侧面装甲 |
| South | 底部中央 | 向下 | 可见车首装甲 |
| West | 左侧 | 向左 | 可见另一侧装甲 |

### 2.3 其他载具类型

#### 半履带车 (Halftrack, e.g., M3/Hanomag)
```
尺寸: 40×44px
特征:
  - 前部车轮(2个圆形, 4px直径)
  - 后部履带(单侧, 6px宽)
  - 开放式货箱(后部矩形, 可见载人)
  - 车载机枪(前部向上)
```

#### 吉普车 (Jeep, e.g., Willys MB)
```
尺寸: 28×20px
特征:
  - 小型轮廓
  - 挡风玻璃V形线条
  - 可见方向盘/座椅轮廓
  - 无厚重装甲(线条较轻)
```

#### 坦克歼击车 (Tank Destroyer, e.g., M10 Wolverine)
```
尺寸: 46×46px
特征:
  - 开顶炮塔 (无舱盖, 可见内部)
  - 长炮管 (超长, 14px)
  - 薄装甲 (线条细)
```

## 四、地形与环境精灵 (Terrain Sprites)

### 4.1 地形Tile (64×64px)

#### 草地 (Grassland)
```
基底: #4C7C23 (深绿)
细节:
  - 3-5根草叶 (2-3px线条, 随机角度)
  - 小花点缀 (1px红/黄点, 10%概率)
  - 土壤斑块 (不规则浅棕区域, 20%tile)
变化: 多种变体(3-5种)避免重复感
```

#### 泥土路 (Dirt Road)
```
基底: #8B6914 (土黄)
细节:
  - 车辙痕迹 (2条平行深色线, 3-4px宽)
  - 石子散落 (1px灰点, 随机分布)
  - 边缘过渡到草地 (羽化边界)
```

#### 碎石路 (Cobblestone)
```
基底: #9E9E9E (灰)
细节:
  - 石块网格 (6×6px不规则多边形, 深灰描边)
  - 缝隙填充 (深灰/黑色, 1px)
  - 个别松动石块 (略偏离网格)
```

#### 河流/运河 (Water)
```
基底: #4080B0 (蓝, alpha=180)
细节:
  - 波光效果 (随机亮白短线, 3-4px)
  - 岸边过渡 (渐变到泥土色)
  - 流动方向 (斜纹暗示流向)
动画: 波纹每2秒平移一次
```

#### 建筑物 (Building)

##### 房屋 (House, 1-2层)
```
尺寸: 占据2×2 tile (128×128px)
屋顶: 红色/灰色梯形或三角形坡顶
墙壁: 米白/浅灰, 有门窗暗框
阴影: 右下角投射 (深色半透明)
破坏状态:
  - 完好: 完整轮廓
  - 损伤: 墙壁裂纹 (黑色不规则线)
  - 废墟: 灰色调 + 瓦砾散落 (随机小方块)
```

##### 教堂 (Church)
```
特征: 尖顶三角形 (高2倍于墙体)
十字架: 顶部小十字 (4×4px)
钟楼: 一侧高起, 圆形钟面
颜色: 白墙/灰顶 或 砖红墙
```

##### 谷仓 (Barn)
```
特征: 大型矩形 (3×2 tile)
屋顶: 双坡陡峭 (60°角)
门: 大型对开门 (底部中央)
颜色: 木质红褐/谷仓灰
```

##### 桥梁 (Bridge)
```
跨越河流的深色条带 (宽8-12px)
桥墩: 水中垂直支撑 (4×6px矩形)
护栏: 两侧细线 (1px)
破坏状态: 断裂缺口
```

### 4.2 装饰物 (Decorations)

```
树木 (Tree):
  - 树冠: 绿色圆形 (16-24px直径, 多层叠加)
  - 树干: 棕色矩形 (4×8px)
  - 阴影: 椭圆形深绿 (半透明)

树篱 (Hedgerow, 诺曼底特色):
  - 密集灌木线 (不规则深绿形状, 高度2-3px)
  - 可阻挡视线和移动

石墙 (Stone Wall):
  - 灰色砖块纹理 (4×4px网格)
  - 不规则高度 (1-2px起伏)

铁丝网 (Barbed Wire):
  - 木桩 (3px高) + 横线 (1px) + 倒刺 (小V形)

弹坑 (Shell Crater):
  - 椭圆形凹陷 (深色填充)
  - 土壤翻起边缘 (亮色环)
  - 中心可能有积水 (蓝点)
```

## 五、UI组件精灵 (UI Component Sprites)

### 5.1 命令按钮图标 (24×24px)

基于用户要求的**7个核心命令**:

| 命令 | 图标描述 | 视觉元素 |
|------|---------|---------|
| Move (前进) | 绿色箭头 | → 或 👣脚印 |
| Fast Move (快速前进) | 双箭头 | ⇒ 或 🏃奔跑人形 |
| Sneak (潜行) | 蜿缩人形 | 🚶 弯腰移动姿态 |
| Attack (攻击) | 红色准星/剑 | ✶ 或 ⚔️交叉剑 |
| Smoke (放烟雾) | 灰色云团 | 💨 云朵形状 |
| Defend/Dig In (防御) | 盾牌/铲子 | 🛡️ 或 ⛏️ |
| Hide/Cover (隐蔽) | 眼睛斜线 | 👁‍🗨 或 🌿灌木 |

### 5.2 状态指示器

```
选中框: 黄色虚线矩形 (围绕单位, 2px宽, 动画闪烁)
目标框: 红色虚线矩形 (敌人, 同上)
移动路径: 绿色虚线 + 箭头
攻击线: 红色实线 (射手→目标)
射程圈: 红色虚线圆环 (最小/最大)
危险区: 红色半透明填充
安全区: 绿色半透明填充
```

## 六、特效精灵 (VFX Sprites)

### 6.1 战斗特效

```
枪口火焰 (Muzzle Flash):
  形状: 椭圆或不规则星形
  颜色: 黄→橙→白 渐变
  尺寸: 6×4px
  持续: 0.05秒 (3帧)

命中火花 (Hit Spark):
  形状: 3-5个小黄/白点散射
  运动: 放射状外扩 (速度2-4px/frame)
  持续: 0.15秒

爆炸 (Explosion):
  核心: 白/黄色圆形扩散 (10→30px)
  烟云: 灰色不规则形 (持续2s, 渐隐)
  碎片: 8-12黑点放射 (重力抛物线)
  屏幕震动: 相机偏移 (2-4px, 0.2s衰减)

血液 (Blood, 低暴力版):
  形状: 红色小点/短线 (2-3px)
  位置: 目标位置
  持续: 0.3s → 渐隐至地面污渍 (持久)
```

### 6.2 环境特效

```
雨滴 (Rain Drop):
  形状: 细蓝色线段 (4-8px长, 1px宽)
  角度: 15-20°倾斜 (风向影响)
  速度: 15px/frame
  密度: 全屏200-400滴

雪花 (Snow Flake):
  形状: 白色圆点 (2-3px直径)
  运动: 下落+左右飘摆 (正弦轨迹)
  速度: 3-5px/frame

烟雾 (Smoke Grenade):
  形状: 灰白色大圆形 (半径3-5格)
  边缘: 不规则锯齿状
  内部: 半透明 (alpha=150-200)
  持续: 3回合 (约45s)
  扩散: 每回合半径+0.5格

火焰 (Fire):
  形状: 橙红色不规则形 (跳动)
  核心: 亮黄色 (高温)
  外围: 暗红色 (低温+烟)
  动画: 3帧循环 (0.2s), 模拟 flicker
```

## 七、制作工具与技术规范

### 7.1 推荐软件
- **Aseprite** (专业像素艺术软件, $20)
- **LibreSprite** (免费开源替代)
- **GraphicsGale** (老牌像素工具)
- **Photoshop** (铅笔模式+网格)
- **PIL/Python** (程序化生成备选)

### 7.2 导出规格
```
格式: PNG (无损压缩, 支持透明)
色彩模式: RGBA (需要透明背景)
色深: 8位/通道 (真彩色)
画布大小:
  - 单位精灵: 32×32 或 48×48 (含padding)
```

## 附录A: 基于2026-05-22 CC2原版截图分析的修正

### A.1 视角修正: 从正俯视改为45°等距投影 (CRITICAL!)

**之前错误**: 假设纯90°上方俯视
**正确方案**: 45° Isometric (伪3D)

#### 视角参数:
```
投影方式: 等距投影 (Isometric Projection)
相机角度: 俯视45° + 旋转30° (或35.264°标准等距)
X轴: 屏幕右下方向 (30°水平)
Y轴: 屏幕左下方向 (150°水平)
Z轴: 垂直向上 (高度)
```

#### 对精灵制作的影响:
```
单位精灵:
  - 不再是纯平面俯视图
  - 需要"假3D"效果:
    * 头部略高于脚部 (Y偏移+2-3px)
    * 身体有轻微厚度 (左右各1px阴影)
    * 武器有透视缩短 (远端略细)

建筑精灵:
  - 必须显示两个可见面 (顶面+一个侧面)
  - 屋顶为菱形/平行四边形 (非矩形)
  - 墙壁有垂直线条表示高度

地形Tile:
  - 从64×64正方形变为 64×32 菱形 (2:1宽高比)
  或保持正方形但内部绘制按等距透视
```

### A.2 尺寸修正: 精灵比预期更小

**截图实测**:

- 树木: 约14×14像素 (圆形树冠)

### A.4 Defend/Dig 命令逻辑修正 (用户要求!)

**重要变更! 用户明确指示:**

#### ❌ 旧设计 (错误):
```
Dig In: 直接在原地挖战壕 (无条件)
Hide: 移动到掩体后
```

#### ✅ 新设计 (正确):
```
Defend (防御):
  优先级1: 扫描周围5格范围内的掩体
    → 树木 (Tree): cover_bonus=+30%, 移动到树后
    → 建筑物 (Building): cover_bonus=+50%, 进入建筑
    → 飞机残骸 (Wreckage): cover_bonus=+40%, 移动到残骸旁
    → 石墙/树篱 (Wall/Hedgerow): cover_bonus=+35%, 靠墙站立
    → 弹坑 (Shell Crater): cover_bonus=+25%, 进入弹坑

  优先级2: 如果范围内无任何掩体
    → 在原地开始挖掘散兵坑 (Dig Trench)
    → 需要3回合 (90 tick) 完成
    → 完成后 cover_bonus=+50%
    → 地面显示壕沟纹理

  行为表现:
    → 有掩体: 快速移动到掩体位置 (1-2格距离)
             采取防御姿态 (蹲下/贴墙)
             保持警戒 (武器指向威胁方向)
    → 无掩体: 原地跪下
             开始挖掘动作 (铲子动画, 0.5s循环)
             进度条显示 (3回合完成)
             完成后进入战壕

Dig In (挖壕 - 仅作为Defend的子选项):
  触发条件:
    a) 手动选择Dig命令 (强制原地挖掘)
    b) 或Defend命令执行时发现无掩体可用

  效果: 同上"无掩体"分支
```

**测试用例更新**:
```python
def test_defend_with_nearby_tree():
    """Defend: 附近有树木→移动到树后"""
    unit = create_unit(pos=(10, 10))
    tree = create_tree(pos=(11, 10))  # 相邻格有树

    assert unit.target_position == (11, 10) or (10.5, 10)  # 移动到树附近
    assert unit.cover_bonus >= 0.30  # 获得树木掩护加成
    assert unit.state == State.DEFENDING

def test_defend_no_cover_digs():
    """Defend: 无掩体→自动挖战壕"""
    unit = create_unit(pos=(10, 10))
    # 周围5格内无任何掩体

    unit.execute_command("defend")

    assert unit.state == State.DIGGING  # 开始挖掘
    assert unit.dig_progress == 0      # 进度0%

    # 模拟3回合后
    for _ in range(90):
        unit.update(1/30)

    assert unit.dig_progress >= 1.0    # 完成
    assert unit.cover_bonus >= 0.50    # 战壕加成
    assert has_trench_marker(unit.position)  # 地面有战壕标记

def test_manual_dig_command():
```
